# Work Spine Boundary

## 1. Purpose
ה-Work Spine הוא תשתית העבודה היומית (Operational Spine) של כלל המערכת. זוהי תחנת הבקרה הניהולית והביצועית המאפשרת לאלתר משימות אמיתיות ולקדמן. מטרתה היא כפיית אמת עכשווית בכל רגע נתון על-ידי אכיפת מעבר סטטוסים מסודר וציון "פעולה הבאה" המעוגן בלוג בקרת שינויים אטום, ללא חריגות למטרות "ניהול ידע".

## 2. Ownership
ה-Work Spine שייך באופן אבסולוטי לספריית `src/work-spine/`. זוהי תשתית רוחבית עצמאית לחלוטין העומדת בפני עצמה וללא ריבונות של שום ישות חיצונית אחרת (כגון Dashboard או הנהלת חשבונות מסורתית).

## 3. Layers
התשתית מקופלת ומיושמת באמצעות 5 שכבות בלתי-תלויות:
- **Domain:** אכיפת השפה העסקית, ההגבלות האפשריות ומילון המונחים (WorkItems, Transitions).
- **Runtime:** פונקציונליות השירות ואוכפי הפעולה במעין רצועות יומן קשיחות (Use Cases שמשדרים גם פעילות לשכבת ה-Audit).
- **Persistence:** רפליקציה פנימית אוטומטית עצמאית לזיכרון התחנה (Repositories מבודדים).
- **UI:** משטח כרטיסיות נקי הקורא וממיר נתונים לעיניים, ובועט בקשות אל ה-Runtime ללא החלטה עצמאית על חוקים.
- **Route / Hosting:** משטח גובה חיצוני בלבד המניח את הלוח במרחב הווירטואלי הרצוי ללא שליטה אדריכלית.

## 4. Physical Location In Code
ההטמעות הפיזיות של החתך מצויות בדיוק במיקומים הבאים:
- UI העבודה המרכזי: `src/work-spine/ui/TodayControlBoard.tsx`
- פקודות ה-Runtime החוסמות ומתניעות הרשאות: `src/work-spine/use-cases/`
- הזרקת המקור לבדיקת התחדשות: `src/work-spine/runtime/work-spine-bootstrap.ts`
- משטח העל המפעיל וחושף אותו החוצה: `src/main.tsx` תחת ה-Route הרשום כ- `/work-spine`

## 5. Explicit Non-Ownership
למען הסר ספק:
- Dashboard יתעדכן ואולי יצמח, אבל **Dashboard או לשכת המנכ"ל לעולם אינם בעלי הבית The Owners** על תחנת העבודה.
- ה-CEO Office אולי ישתמש במדדים, אך **אין לו בעלות דומיינית כאן**.
- למרות חיבור לוגיקת קבצי חשבונאות, ה-**Accounting Core** היא ישות אחות או בת, **ולא הבעלים**.
- שום **Page Domain אחר (כגון שירותי Onboarding, דפי נחיתה)** אינם משתלטים או בעלים פה. ה-Work Spine נמצא בקנה מידה גלובלי.

## 6. Hosting Rule
ה-router של האפליקציה (כמו שכבת `<Layout />` או `<Routes>`) מתפקד כשכבת השליחות הגבוהה בלבד, הנותנת מקלט והוראות גישה לתחנת ה-Work Spine (משמש כ"מארח"). הראסטה הזו מציגה את התחנה לכל פועל רלוונטי, אך אין מדובר ביכולתה להגדיר בשום קונסטלציה בעלות לוגית על תוכן הפקודות.

## 7. Final Rule
Work Spine הוא תשתית עבודה רוחבית עצמאית, והוא רק מקבל Hosting משכבת הראוטר של המערכת.
